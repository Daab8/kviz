AI Quiz JSON Generation Guide (concise)

Purpose
- Short reference for generating quiz JSON that the app accepts. Follow schema, validation rules, and generation constraints below.

Root structure
- The top-level JSON must include: `version`, `categories` (array), `questions` (array), `quizzes` (array).
- Minimal wrapper example:
  {
    "version":"1.0.0",
    "categories":[],
    "questions":[],
    "quizzes":[]
  }

Question object (summary)
- `id` (string, required): unique id, stable.
- `text` (string, required): question text.
- `answerType` ("single"|"multiple"|"ordering", required).
- `points` (integer, required): set to `1` for every question.
- `answers` (array, required for `single` and `multiple`, recommended for `ordering`): exactly 4 items labeled `A`,`B`,`C`,`D`.
  - answer object: `id` (A..D), `text` (string), optional `correct` (boolean), optional `explanation`, optional `imageDataUrl` or `imageUrl`.
- `ordering` (array of A..D) required for `answerType: "ordering"` (correct sequence).
- `tags` (array), `description` (explanation shown on reveal), `meta` (optional).
- `shuffleAnswers` (boolean, optional) — presentation hint; defaults to `true`.

Quizzes and categories (summary)
- `quizzes`: objects with `id`, `name`, and optional `selectedQuestionIds`, `selectedCategoryIds`, `orderedQuestionIds`.
- `categories`: `id`, `name`, `description`.

Examples (minimal)
- Single-answer (one `correct`):
  {"id":"q-001","text":"Which is red?","answerType":"single","points":1,
   "answers":[{"id":"A","text":"Apple"},{"id":"B","text":"Banana"},{"id":"C","text":"Cherry","correct":true},{"id":"D","text":"Grape"}],"description":"Cherry is red."}
- Multiple-answer (2..4 correct):
  {"id":"q-010","text":"Select primes","answerType":"multiple","points":1,
   "answers":[{"id":"A","text":"2","correct":true},{"id":"B","text":"3","correct":true},{"id":"C","text":"4"},{"id":"D","text":"5","correct":true}]}
- Ordering (include `ordering`):
  {"id":"q-020","text":"Order events","answerType":"ordering","points":1,
   "answers":[{"id":"A","text":"Event A"},{"id":"B","text":"Event B"},{"id":"C","text":"Event C"},{"id":"D","text":"Event D"}],"ordering":["A","B","C","D"]}

Key generation rules (compact)
- Mix topics: interleave categories so players don't see many same-topic questions in a row.
- IDs & labels: use single uppercase letters `A`..`D` for answer ids; unique per question.
- Avoid leakage: do not include explicit years, ordinals, or unique tokens in answer texts that reveal the solution — move such metadata to `description`.
- Plausible distractors: keep distractors same category/length/register as the correct answer.
- Counts & formatting: exactly 4 answers per question; `single` → exactly 1 `correct:true`; `multiple` → 2..4 correct; `ordering` → use `ordering` array of all A-D.
- Points & difficulty: always `points: 1`; aim for at least high-school level difficulty.

Images (short)
- Prefer embedding as `imageDataUrl` (data:<mime>;base64,...) for reliable display. Maker UI limits uploads to ~512 KB.
- If using `imageUrl`, the app will not auto-fetch it — convert to `imageDataUrl` before import if you need immediate display.
- Quick conversion: download binary → detect MIME → base64 encode → build `data:<mime>;base64,<payload>`.

Ordering-specific
- `ordering` must list all four IDs once (e.g., `["A","B","C","D"]`).
- Set `shuffleAnswers: true` in the question source so displayed order can be randomized at runtime.

Consistency checks (apply before export)
- Unique question ids; exactly 4 answers (A..D) each; points == 1.
- `single` has one correct; `multiple` has 2..4 correct; `ordering` has ordering array that contains A-D once.
- No duplicate answer texts; no answers repeating the full question.

Prompt template (short)
- "Generate N Slovak questions for TOPIC. Output JSON with `categories`, `questions`, `quizzes` following the above schema. Ensure 4 answers per question, correct flags, `ordering` for ordering-type, and `points:1`. Avoid hints or dates in answers."

Checklist (quick)
- IDs unique; answers A..D; points=1; counts/answerType rules met; no hint leakage; parseable JSON.

Slovak-language requirements
- All `text`, `description`, and `answers` must be in Slovak for imports intended for this app. Keep phrasing natural and idiomatic.

Distributed generation (note)
- If splitting work across workers, ensure global ID uniqueness (use prefixes), then merge and run the full validation checklist.

Implementation notes for integrators
- Maker UI normalizes/uppercases ids and strips any `shuffleAnswers` flag on import; the app expects answers in A..D format.
- Images embedded as `imageDataUrl` are safest; external `imageUrl` may not display without conversion.

End of concise guide
